## 1. Client ask (verbatim)
Persona: Dr. Maya Okonkwo, COO.
> Hi, and welcome to Riverbend! … our front desk keeps saying the registration page 'spins' for
> ages, and honestly the board has been on me to do *something* with AI — could you stand up a
> little assistant that drafts a patient-friendly version of our intake instructions? … the logins
> are all in there so you can just run it. We're fully HIPAA compliant already, so don't worry
> about that side — just get something on the screen this week.

## 2. What it really is
- "Fully HIPAA compliant already" is **false** — intake logs full request bodies (PHI) at INFO, PHI
  stored plaintext, secrets in git history. The AI assistant is the vehicle; the real work is a
  **PHI-safe LLM wrapper** (timeout / retry / structured output / token-cost guard) + a redaction
  layer + a debt log recording what's actually non-compliant.

## 3. Scope
### In scope
- LLM wrapper with: request timeout, bounded retry, structured-output parsing, token/cost guard.
- PHI **redaction** on anything that leaves the app boundary (logs, prompts).
- Debt log documenting the false-compliance claim and the D1 gap.
### Explicitly OUT (owning week)
- Provider abstraction / Bedrock migration — shipped later, not the W1 line.
- Cloud tracing / observability → W7.
- Safe-Harbor 18-identifier de-id + BAA governance → W8.
- Fixing intake's synchronous eligibility freeze → W3 (RIV-088/141).

## 4. Deliverables
1. LLM wrapper module (timeout/retry/structured/token-guard).
2. PHI redaction function + policy doc.
3. Debt log entry (D1) + honest compliance note.
4. ADR if a non-trivial design decision is made.

## 5. Acceptance criteria (done-definition)
- [ ] Wrapper enforces a hard timeout and bounded retry; proven by test.
- [ ] Wrapper returns validated structured output (schema), not free text.
- [ ] Token/cost guard caps a runaway call; proven by test.
- [ ] Redaction masks PHI **in every field**, including non-PHI keys and list items — end-to-end
      scan test exists (CLAUDE.md §5 negative-test rule; the `consents` leak lesson).
- [ ] At least one **adversarial** redaction test (PHI planted where code doesn't expect it).
- [ ] Debt log records D1 and flags the false "already compliant" claim.

## 6. Landmines (CLAUDE.md §6)
- ⚠️ Intake logs full bodies (PHI) at INFO — do not copy this pattern.
- ⚠️ Plaintext PHI (`ssn`, `notes`).
- ⚠️ Secrets in git history; no secret/vuln scan in CI — do not add more secrets; flag before rotating.
