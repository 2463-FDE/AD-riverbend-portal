# W6 — HL7 allergy / med mapping fix

> scope / deliverables / acceptance criteria are ours.

- **Status:** derived by `/dashboard` from live repo state — not stored here (see §5).

- **Tickets:** RIV-160 (allergy differs per chart — HL7-drop facet)
- **Deliverable type:** legacy-comprehension report + characterization tests + failing test + ADR

## 1. Client ask (verbatim)
> We're finally wiring up the feed from the hospital's system — it sends us those HL7 messages. The
> integration 'works' in that data shows up, but before we lean on it for care I want to actually
> trust it. Can you go through the parser and make sure it's solid? … It's been quietly running for
> months with zero errors, which is reassuring.

## 2. What it really is
- **"Zero errors" is the tell.** The parser silently **drops AL1 (allergy) and RXA (medication)**
  segments instead of erroring — only PID/PV1 are mapped. Missing allergies is a patient-safety issue.
  Deliverable: legacy-comprehension report + characterization tests + a **failing test demonstrating
  the drop** + an ADR proposing schema-validated mapping.

## 3. Scope
### In scope
- Comprehension report of the current parser (what it maps, what it silently discards).
- Characterization tests capturing current behavior (PID/PV1 mapped, AL1/RXA dropped).
- A **failing** test that demonstrates the AL1/RXA drop (the defect, made visible).
- ADR proposing schema-validated mapping that errors instead of dropping.
### Explicitly OUT (owning week)
- The dup-patient / MPI facet of RIV-160 → W2.
- Full re-implementation of the parser — ADR proposes; impl is future.

## 4. Deliverables
1. Legacy-comprehension report.
2. Characterization tests (green — current behavior).
3. Failing test demonstrating AL1/RXA drop.
4. ADR: schema-validated mapping (error/quarantine, don't silently drop).

## 5. Acceptance criteria (done-definition)
- [ ] Report identifies every segment the parser reads vs. silently discards.
- [ ] Characterization tests pin PID/PV1 happy path (green).
- [ ] A test **fails** today proving AL1 (allergy) and RXA (med) are dropped — matches the existing
      `xfail`; do not "fix" the test to hide the gap (CLAUDE.md §5).
- [ ] ADR proposes schema validation that surfaces an error/quarantine instead of a silent drop,
      with the patient-safety rationale.

## 6. Landmines (CLAUDE.md §6)
- ⚠️ Brittle HL7 mapping — only PID/PV1 mapped; AL1/RXA silently dropped (RIV-160).
- ⚠️ HL7 allergy/med extraction is an `xfail` in the suite — characterize, don't hide.
