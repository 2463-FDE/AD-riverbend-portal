## 1. Client ask (verbatim)
> A payer audit is coming and I want us buttoned up. Tighten the security generally, and — staff keep
> asking — can we finally add the records-request (ROI) feature so we stop emailing PDFs around?
> Everyone here logs in with their own account, so access control is basically handled; I just want
> it to look clean for the auditor.

## 2. What it really is
- **"Access control is basically handled" is false.** Single `staff` role, no segregation of duties,
  no MFA; and ROI has **zero 45 CFR 164.508 authorization enforcement** and no accounting of
  disclosures. Deliverable: RBAC/least-privilege **spec** + ROI **ADR** (164.508 authorization +
  164.522 restriction honoring) + MFA **recommendation** — design-level, not full build.

## 3. Scope
### In scope
- RBAC / least-privilege spec (roles beyond the single `staff`, per-action authz, SoD).
- ROI ADR: 45 CFR 164.508 authorization enforcement + 164.522 restriction honoring + disclosure accounting.
- MFA recommendation.
### Explicitly OUT (owning week)
- Full RBAC / ROI **implementation** — spec/ADR only.
- The append-only audit trail + disclosure-accounting *report generator* → W10 (this week designs
  the authz; W10 builds the trail).
- IDOR patient-binding → W4.

## 4. Deliverables
1. RBAC / least-privilege spec.
2. ROI ADR (164.508 + 164.522 + accounting).
3. MFA recommendation.

## 5. Acceptance criteria (done-definition — for the spec/ADR artifacts)
- [ ] Spec refutes "access control is basically handled" with the single-role / no-SoD / no-MFA evidence.
- [ ] RBAC spec defines concrete roles + per-action authz mapped to the real routes (front desk,
      clinician, ROI clerk, scheduler).
- [ ] ROI ADR requires a recorded 164.508 authorization before any disclosure and defines the
      accounting-of-disclosures record.
- [ ] ROI ADR addresses 164.522 (honoring patient-requested restrictions).
- [ ] MFA recommendation states mechanism + rollout, not just "add MFA."

## 6. Landmines (CLAUDE.md §6)
- ⚠️ Auth / sessions — single role, no MFA, sessions never expire. Never change auth behavior without
  explicit human approval (spec/ADR is safe; impl is not, this week).
- ⚠️ ROI has no authorization enforcement — disclosures go out with no recorded 164.508 authorization
  and no accounting trail.
